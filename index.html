📚 대시보드 사용 설명서

이 대시보드는 자사(육회바른연어) + 모니터링 대상 9개 (4-Tier) YouTube 동향을 한눈에 보기 위한 도구입니다. 매일 06:00 / 18:00 자동 업데이트, 누적 데이터를 관리합니다. 상단 `📊 YYYY-MM-DD ~ YYYY-MM-DD` 에서 실제 수집 범위와 마지막 업데이트 시각을 확인하세요.

🏆 순위 (조회수 기준)

경쟁 10개 브랜드 중 총 YouTube 조회수 기준 순위. 1위 = 조회수 가장 많음.

해석 주의:
  • T1 직접 경쟁사·T2 벤치마킹 브랜드는 광고비로 만든 조회수가 대부분 (유튜브 광고 노출 = 조회수 카운트)
  • 자사가 하위에 있어도 "약하다"가 아니라 광고 투자를 덜 했다는 의미
  • 실제 자발 관심은 🎤 Earned SOV가 더 정확 (벤치마크 테이블에서 확인)

🎯 PESO 프레임워크 (3층 사용)

마케팅 업계 표준 PESO (Paid·Earned·Shared·Owned) 4분류. 본 대시보드는 그 중 3가지만 사용합니다 — 이유는 아래에서 설명.

✅ 사용 중인 3종:
  • 🏛 Owned — 자사 공식 채널
  • 🤝 Sponsored (Paid 변형) — 크리에이터 협찬 영상
  • 🎤 Earned — 자발 크리에이터 콘텐츠
❌ 빠진 1종 — Shared:

전통 PESO에서 Shared = "사용자가 공유·좋아요·댓글로 재확산하는 것". 하지만 본 대시보드에선 별도 카테고리로 분리하지 않았습니다.

제외 이유 3가지:
  • ① 이중 집계 방지 — Shared 를 별도 집계하면 Owned/Earned 영상에서 발생한 좋아요·댓글을 2번 카운트하게 됨 (예: 타사 공식 광고의 좋아요 → Owned & Shared 양쪽)
  • ② 이미 💬 Engagement SOV 로 측정 중 — KPI 5번째 카드의 "좋아요+댓글 시장 점유" = 전통 Shared 개념과 사실상 동일
  • ③ YouTube 특성 — YouTube 의 "공유" 지표(Share count)는 yt-dlp 로 수집 불가. 좋아요·댓글만 가능

→ 결론: Shared 는 Engagement SOV 카드로 대체. 3+1 구조 (PESO 3분류 + Engagement) 로 운영.

PESO 의 각 층 의미:

층의미예시자사(육바연)가 쓸 수 있는 레버
🏛 Owned 자사 공식 채널이 업로드한 영상 육회바른연어 · 자사 공식 채널 광고 광고 제작 · 예산 집행
🤝 Sponsored 크리에이터가 돈받고 만든 협찬 영상 #광고 · "유료광고 포함" 표시가 있는 영상 크리에이터 협찬 집행
🎤 Earned ⭐ 크리에이터가 자발적으로 만든 영상 일반 먹방·리뷰·뉴스 영상 제품 자체 화제성 (자사 주력)
KPI 카드 읽는 법:
  • 큰 숫자 = 선택된 브랜드 내 비중 (Owned + Sponsored + Earned = 100%)
  • 회색 배지 = "시장 OO 중 X% 점유" (경쟁사 대비 점유율)
  • 예: 자사 내 Earned 99.6% = 자사 영상의 99.6%가 크리에이터 자발 / 시장 Earned 중 21.9% = 전체 Earned 시장의 21.9% 차지

🎭 감성 분석 파이프라인

감성 분류는 KR-BERT + Qwen3 하이브리드 구조입니다. 대량 댓글은 BERT 가 빠르게 1차 분류하고, BERT 가 '부정'으로 판정한 댓글 중 워닝 후보만 Qwen3:4b LLM 이 2차 검증해 YouTube 특유의 아이러니·밈·타임스탬프 코멘트를 교정합니다.

🔬 2단계 파이프라인:
  1. 1차 — KR-BERT (한국어 리뷰 학습, 로컬 MPS GPU, ~수천 댓글/분): 제목+설명(내용 감성), 상위 50개 댓글(댓글 감성) 각각 긍정/중립/부정 3분류
  2. 규칙 오버라이드 (아래 "자동 교정 규칙" 참조): 슬랭·밈·웃음·질문·타임스탬프 등 한국어·YouTube 특유 패턴 교정
  3. 2차 — Qwen3:4b (Ollama 로컬, ~4 GB RAM, 배치 10건): BERT 가 '부정'으로 남긴 댓글 중 워닝 후보 영상(부정률 20%↑)의 것만 LLM 재검증 → 아이러니·웃음·질문을 재분류

각 영상 카드에 2가지 감성 배지가 병렬 표시됩니다:

배지의미신뢰도

📝 😊 내용 긍정 내용 감성: 제목+설명 KR-BERT + 규칙 오버라이드. 크리에이터 의도 추정 ★ (보조 지표)

  • 멀티브랜드 영상은 경고 등급 한 단계 감쇠 (크리에이터 팬 댓글 신뢰도 ↓)

댓글 수집 한도: 경쟁사 영상당 100개 · 대시보드 노출 상위 50개 (좋아요 기준). 15+ / 30+ 임계는 통계적 안정성을 위한 최소값입니다.

🧹 브랜드 무관 영상 자동 스크리닝

제목에 브랜드명이 있어도 실제 내용이 무관하면 자동 제외합니다. 수집 파이프라인과 대시보드 양쪽에서 다단 필터.

판정 우선순위:
  1. 브랜드별 명시 제외 리스트 (동음이의 일반어 (박수치다·달맞이꽃·풍자(문학) 등))
  2. 공식표기 필수 — 짧은 별칭만으로는 통과 불가. 공식 표기 또는 시그니처 메뉴(육회연어국수·빠삭파전·켄치밥 등) 필수
  3. 채널명 엄격 모드 — 동음이의 함정이 있는 채널은 묶음 토큰만 통과 (예: "풍자 또간집")
  4. 박수식당 컨텍스트 필수 — "박수" 있어도 식당/육회/사직/야구장 등 동반 없으면 제외
  5. 밈·비교 패턴 — "브랜드랑 같은", "브랜드처럼" + 맥락 부족 → 제외
    예: "박수받았다" (응원/갈채 의미, 식당과 무관)
  6. 컨텍스트 마커 검증 — 리뷰/먹방/매장/메뉴/시그니처 있으면 통과
  7. 댓글 교차 검증 — 댓글 내 브랜드 언급률 < 50% 이면 제외
주요 차단 동음이의 사례:
  • 서브웨이 — 서브컬처/서브에이전트/서브티처/서브스터디/탁구·테니스·배드민턴 서브/마라톤 서브3·서브4/서브캠/서브우퍼/서브퀘스트/서브노티카(게임)/지하철·전철/Subway Serenade/Today's Stockings
  • BBQ — 텍사스 바베큐/Korean BBQ/K-BBQ/삼겹살·오겹살·꽃갈비·꽃등심·살치살·토시살/마블링·드라이에이징/훈제·스모커/양꼬치/BBQ체어·전기그릴/게임방송(템진단·똥꼬쇼·룰렛)/라이브 음악 비비큐
  • 맥도날드 — '맥도' 단독 제거 (맥도유저/맥도아이디 = 게임 유저명, 맥도생태공원 = 지명). 시그니처 메뉴(빅맥·맥너겟·맥치킨·맥스파이시·맥카페·맥모닝) 필수. 맥주/맥북/맥프로/심법/붉은사막/리니지 등 제외
  • 롯데리아 — 롯데그룹/롯데쇼핑/롯데마트/롯데월드 등 계열사 뉴스 제외
  • 교촌 — 교촌한옥마을(경주 지명) 제외
수집 단계 차단: 서브웨이 검색 쿼리에서 '서브'·'써브' 짧은 별칭 제거. 수집 게이트에서 공식표기 없으면 CSV에 저장되지도 않음. CSV 비대화 방지.

❓ 자주 묻는 질문

Q. 자사 순위가 낮게 나오는데 잘하고 있는 게 맞나요?

기본 순위는 총 조회수 기준인데 광고 집행이 큰 브랜드가 유튜브 광고로 수천만 조회를 찍어서 상위권. 하지만 자발 크리에이터 관심(🎤 Earned)에서는 자사는 광고 의존도가 낮은 자발적 화제성 브랜드라 Earned 비율이 높음. 광고비 투자 부족이 순위에 반영된 것일 뿐.

Q. 단일 광고 한 편이 수천만 뷰가 말이 되나요?

YouTube 인스트림/인피드 광고는 사용자가 광고 건너뛰기 전 잠깐 본 것도 조회수 1로 카운트합니다. TV CF + 유튜브 광고에 수억 투입하면 2천만 뷰 달성은 현실적입니다.

Q. 부정률이 왜 이렇게 들쑥날쑥한가요?

특정 브랜드가 갑자기 부정 비율 높아지면 사건사고 가능성. 일반적으로 자사·T1·T2는 호의적 리뷰가 다수. 한국어 슬랭 (폭력적=최고, 죽인다=최고) 자동 교정 적용 후 현실 반영.

Q. Sponsored 영상이 거의 없는데 왜 생겼나요?

현재 감지는 제목+설명 300자 범위에서 "#광고", "유료광고 포함" 같은 명시적 마커만 잡습니다. 미공개 협찬(20~30% 추정)은 Earned로 오분류될 수 있음. 향후 자막 기반 고도화 예정.

Q. 어떤 채널이 항상 0%인 이유?

공식 채널 미운영(예: 박수식당) 또는 자사 신규 브랜드는 초기 데이터가 적음. 크리에이터 자발 영상도 희박해서 실제 "데이터가 없는" 상태입니다.

Q. 언제 데이터가 업데이트되나요?

매일 06:00 (아침)과 18:00 (저녁) 자동 크롤. Slack 발송은 08:30 (아침만). 대시보드는 브라우저 하드 리프레시(⌘+Shift+R)하면 최신 버전 반영됩니다.

Q. 데이터는 어디서 오나요?

yt-dlp 오픈소스 도구로 YouTube 검색·공식 채널 크롤. YouTube Data API 미사용 (토큰·쿼터 없음). 감성 분석은 로컬 KR-BERT, 요약은 로컬 Ollama qwen2.5:14b. 외부 API 비용 0원.
